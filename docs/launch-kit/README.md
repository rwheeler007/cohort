## Five Positioning Pillars

1. **Category**: "AI Team Coordination" -- we created it, we define it
2. **Differentiator**: "Finished" -- every competitor gives drafts, we give finished work
3. **Anti-Disruption**: UX familiarity (zero learning curve) + IBM Principle (safe bet that performs)
4. **Security-First**: Honest risk posture, security agent in every workflow, human approval gates
5. **Day One Productive**: Zero friction to value across all segments

## Core Messages (Use Everywhere)

**One-liner:** "ChatGPT is a person. Cohort is a company."

**Tagline:** "Same processes. Same standards. Same discipline. Just dramatically more capacity."

**Force multiplier:** "A force multiplier for the people and processes you've already invested in."

**Security:** "Security isn't a feature we added. It's how we built."

**Anchor metaphor:** The hammer vs. welding rig (full version in positioning doc).

## Three-Layer Rule

Never mix these in the same context:

- **Billboard** (everyone): Business outcomes. "Your AI team, thinking together."
- **Spec Sheet** (engineers): Technical credibility. "Zero-dep, protocol-first."
- **Proof Story** (skeptics): Concrete demo. "One sentence -> 7 agents -> finished code."

## Hard Rules

- Never reference competitor security incidents by name or implication
- Never name-drop Slack/Teams in permanent brand copy (use in demos/sales only)
- Never claim agent counts that don't match what ships with PyPI
- Never claim "enterprise security," SOC 2, or formal audit status we don't have
- Never use padlock/shield/checkmark symbols for security -- show architecture flow
- IBM Principle is enterprise-segment specific -- don't put on homepage hero
- "Day One Productive" needs cold-user validation before becoming a permanent tagline
