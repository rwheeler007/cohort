    Commands

    cohort gateFree
    Should this agent respond? Runs the full contribution scoring engine and returns a SPEAK/SILENT decision with the score breakdown. Exit code 0 = speak, 1 = don't. Pipe into your orchestration scripts.
    # Check if the architect should respond
    $ python -m cohort gate --agent architect --channel review --file conv.jsonl --agents agents.json
    Agent: architect
    Score: 0.73
    Threshold: 0.45 (ACTIVE)
    Decision: SPEAK
    --agent
    Agent ID to check (required)
    --channel
    Channel ID (required)
    --file
    Path to conversation .jsonl file (required)
    --agents
    Path to agents.json config (required)
    --format
    Output format: json | text (default: text)
    cohort next-speakerFree
    Who should talk next? Ranks all agents by composite relevance score -- domain match, recency, phase detection, and stakeholder status. Returns the top N candidates.
    $ python -m cohort next-speaker --channel review --file conv.jsonl --agents agents.json --top 3
    Next speaker for channel 'review':
    1. security_agent score=0.82 phase=review (domain=0.45, recency=0.20)
    2. python_developer score=0.71 phase=review (domain=0.38, stakeholder=0.18)
    3. qa_agent score=0.64 phase=review (domain=0.30, recency=0.15)
    --top
    Number of speakers to show (default: 3)
    --format
    json | text
    cohort sayFree
    Append a message to a conversation. Auto-creates the channel if it doesn't exist. Use this to script agent interactions or inject messages programmatically.
    $ python -m cohort say --sender architect --channel review --file conv.jsonl --message "Let's review the auth module"
    [OK] msg_a1b2c3 -> #review
    cohort briefingFree
    Generate or view executive briefings. Summarizes channel activity, task progress, agent performance, and blockers over a configurable time window. Posts to a briefing channel by default.
    # Generate a new briefing covering the last 24 hours
    $ python -m cohort briefing generate --hours 24
    # View the latest briefing as JSON
    $ python -m cohort briefing latest --format json
    generate --hours N
    Hours to cover (default: 24)
    generate --no-post
    Don't post to channel, just print
    generate --data-dir
    Data directory (default: data)
    latest
    Show the most recent briefing
    cohort setupFree
    Interactive 8-step setup wizard. Detects your hardware (GPU, VRAM, platform), installs Ollama, pulls the right model for your GPU, configures the MCP server for Claude Code, and sets up content feeds. Uses only stdlib -- no extra pip dependencies.
    $ python -m cohort setup
    Step 1/8: Detecting hardware...
    GPU: NVIDIA RTX 3080 (12 GB VRAM)
    Recommended model: qwen3.5:9b (6.6 GB)
    [OK] Hardware detection complete
    VRAM-based model selection: 0-4 GB -> 2B model, 4-6 GB -> 4B, 6-10 GB -> 9B, 10+ GB -> 9B with headroom. CPU-only mode supported.
    cohort tutorialFree
    Interactive walkthrough of Cohort's scoring engine. Step through the gate check and contribution scoring algorithms with real examples. Learn how agents decide who speaks when.
    $ python -m cohort tutorial

    Scripting Example

    Combine CLI commands to build custom orchestration pipelines. The JSON output mode makes it easy to pipe into other tools.

    #!/bin/bash
    # Auto-moderate: find who should speak, then gate-check them
    NEXT=$(python -m cohort next-speaker \
    --channel review --file conv.jsonl \
    --agents agents.json --top 1 --format json)
    AGENT=$(echo "$NEXT" | jq -r '.[0].agent_id')
    if python -m cohort gate --agent "$AGENT" \
    --channel review --file conv.jsonl --agents agents.json; then
    echo "$AGENT should respond"
    fi
