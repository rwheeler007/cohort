Installation

Cohort's core library has zero dependencies -- just the Python standard library. Install extras only for the features you need. Requires Python 3.11+.

Install options
# Zero-dependency core
pip install cohort

# REST API + Socket.IO dashboard
pip install cohort[server]

# MCP bridge for Claude Code
pip install cohort[claude]

# Agent-as-a-service HTTP API
pip install cohort[agent-api]

# Everything
pip install cohort[all]
[QS]

Quick Start

Create a channel, define agents, start a session, and let the orchestrator pick who speaks next. This is the complete working example -- 10 lines to your first multi-agent meeting.

examples/quick_cohort.py
from cohort import JsonFileStorage, Orchestrator
from cohort.chat import ChatManager

# 1. Set up storage and chat
chat = ChatManager(JsonFileStorage("my_data"))
chat.create_channel("design-review", "API design review")

# 2. Define agents
agents = {
    "architect": {"triggers": ["api", "design"], "capabilities": ["backend architecture"]},
    "tester":    {"triggers": ["testing", "qa"], "capabilities": ["test strategy"]},
}

# 3. Start orchestrator session
orch = Orchestrator(chat, agents=agents)
session = orch.start_session("design-review", "REST API design review", list(agents))

# 4. Get next speaker recommendation
rec = orch.get_next_speaker(session.session_id)
print(f"Next: {rec['recommended_speaker']} - {rec['reason']}")

# 5. Record a turn (agent spoke)
orch.record_turn(session.session_id, "architect", "msg-001")

No HTTP server required. The orchestrator works directly with file-backed storage. Add cohort[server] when you want the dashboard.

[AG]

Agent Development

Agents are duck-typed. Any object with name, role, capabilities, relevance_score(), and can_contribute() satisfies the protocol. No base class, no imports, no vendor SDK.

cohort/registry.py -- the entire agent interface
@runtime_checkable
class AgentProfile(Protocol):
    """Any object with these attributes works."""

    name: str
    role: str
    capabilities: list[str]

    def relevance_score(self, topic: str) -> float: ...
    def can_contribute(self, context: dict) -> bool: ...

For most use cases, define agents as simple JSON. Cohort's built-in AgentConfig handles scoring automatically.

agents.json -- agent definition
{
  "architect": {
    "triggers": ["api", "design", "architecture"],
    "capabilities": ["REST API design", "system design"],
    "complementary_agents": ["developer", "tester"],
    "phase_roles": {
      "DISCOVER": "high",
      "PLAN": "high",
      "EXECUTE": "low"
    }
  }
}

The orchestrator scores agents across 5 dimensions to pick the next speaker:

Domain Expertise
25%
Complementary
20%
History
15%

10%
Data Ownership

Agents progress through stakeholder statuses (active -> approved_silent -> observer -> dormant) as their contribution threshold rises. Topic shifts re-engage dormant agents whose expertise becomes relevant again.

Storage Backends

Storage is protocol-based. Cohort ships two backends; implement the interface to bring your own (Postgres, Redis, SQLite, etc.).

cohort/registry.py -- storage protocol
class StorageBackend(Protocol):
    def save_message(self, channel: str, message: dict) -> str: ...
    def get_messages(self, channel: str, limit: int = 50) -> list[dict]: ...
    def save_channel(self, channel_id: str, metadata: dict) -> None: ...
    def get_channel(self, channel_id: str) -> dict | None: ...
    def list_channels(self) -> list[dict]: ...
    def delete_channel(self, channel_id: str) -> bool: ...
    def restore_channel(self, channel_id: str) -> bool: ...
JsonFileStorage
Flat JSON files. Default backend. Messages in messages.json, channels in channels.json. Soft-delete with 30-day trash retention.
JsonlFileStorage
Append-only JSONL. One line per message. Ideal for polyglot agent pipelines where multiple processes write concurrently.
[CLI]

CLI Reference

Cohort works without a server. Use CLI commands for file-based scoring, polyglot integration, and quick experiments.

CLI commands
# Start HTTP server with dashboard
python -m cohort serve --host 0.0.0.0 --port 5100 --data-dir data

# Post a message (file-based, no server)
python -m cohort say --sender architect --channel review \
    --file conv.jsonl --message "We need pagination"

# Should this agent speak? (exit 0 = yes, 1 = no)
python -m cohort gate --agent developer --channel review \
    --file conv.jsonl --agents agents.json --format json

# Who should speak next? (ranked list with scores)
python -m cohort next-speaker --channel review \
    --file conv.jsonl --agents agents.json --top 3

# Generate executive briefing
python -m cohort briefing generate --hours 24 --format json

# Interactive setup (Ollama + models + feeds)
python -m cohort setup

The gate and next-speaker commands work with JSONL files, making Cohort a drop-in scoring engine for agents written in any language -- Python, Node, Go, Bash scripts.
