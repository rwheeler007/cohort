Frequently Asked Questions

What hardware do I need?

A single consumer GPU with 8-12GB VRAM runs the default model (qwen3.5:9b) comfortably. An RTX 3060/3070/3080 or equivalent is plenty. CPU-only mode works but is slower. No cloud GPU required.

Is the free tier actually usable in production?

Yes. The open source tier is the same orchestration engine that runs in Pro and Enterprise. Unlimited agents, full roundtables, contribution scoring, work queue, persistent memory. The paid tiers add pre-built vertical pipelines (website builder, content engine, briefing generator) and support -- they don't gate the core framework.

Can I use cloud models instead of local ones?

On the Pro and Enterprise tiers, yes. Cohort supports cloud model fallback to Claude, GPT, and other providers as an escalation path. The open source tier runs entirely on local models -- you supply the GPU, Cohort supplies the orchestration.

What does "protocol-first" mean for vendor lock-in?

Cohort uses duck-typed Python protocols for agents, storage, and inference. Implement the interface and Cohort uses it. No base classes, no vendor SDK. pip install cohort has zero dependencies beyond the standard library. If you stop paying, your agents, data, and workflows remain fully functional on the open source tier.

How does the 14-day trial work?

Full Pro features for 14 days. No credit card required to start. If you don't upgrade, you keep the open source tier with no data loss -- your agents and their accumulated knowledge stay intact.

Do you offer annual pricing?

Yes. Annual Pro is $470/year (2 months free). Enterprise pricing is negotiated per contract. Contact us for details.
